ServeMux 是 Go 标准库中的 HTTP 请求多路复用器（路由器），主要作用是：

根据请求的 URL（包括方法、主机名和路径），将请求路由到对应的处理函数（handler）。 

核心功能简述：
1. 模式匹配
支持注册带路径、方法、主机名的路由模式，例如：
"/api/" → 匹配所有以 /api/ 开头的路径
"GET /health" → 仅匹配 GET 请求
"example.com/" → 仅匹配该域名的请求
支持路径变量："/users/{id}"，可通过 r.PathValue("id") 获取值
2. 优先级匹配
当多个路由匹配时，选择**最具体（最精确）**的那个；冲突的路由注册会 panic。
3. 自动重定向
对于带尾部斜杠的子树路由（如 /static/），访问 /static 会自动 301 重定向到 /static/（可覆盖）。
4. 安全净化
自动清理 URL 中的 .、..、重复斜杠等不安全路径，并标准化 Host 头。
5. Go 1.22+ 行为变更
路由语法更严格（支持真正的路径变量、分段解码等），旧行为可通过 GODEBUG=httpmuxgo121=1 恢复。
